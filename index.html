Chaining vs. Chainless

Before reading the explanation individual commands it is important to understand the difference between chained and chainless d3 commands.

Chained commands use a dot and then the command name to connect to other commands. In this format, order matters.

Chainless commands use variables to assign commands to. Once those variables have been made they can bee strung together like normal javascript. In this format order only matters when combining variables, but not when defining them.

The Select Command

The select command is used to do choose a certain CSS selector.
It's syntax is 

d3.select("body")

The Append Command

The append command is used to create any new element the user specifies, within the DOM element the user selects.
It's syntax in context is 

d3.select("body").append("p")

The Text Command

The text command is used to insert a string into the currently selected element.
It's syntax in context is 

d3.select("body").append("p").text("Let's hear a story all about how my life got flipped, turned upside down . . .");

The Style Command

The style command is used to change or add CSS properties.
It's syntax in context is 

d3.select("body").append("p").text("And I'd like to take a minute. Just sit right there.I'll tell you how I became the prince of a town called Bel Air . . .").style("color", "red");

d3.select(".example") .append("p") .html("Click me if you're fresh") .style(font-size:25px;)
